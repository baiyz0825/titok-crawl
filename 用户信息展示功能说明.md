# 用户信息展示功能说明

## 📋 功能概述

登录成功后，前端登录页面会自动展示当前登录用户的详细信息，包括：
- 用户头像
- 昵称
- 抖音号
- 用户ID（sec_user_id）
- 个人简介
- 关注/粉丝/获赞数据

## 🎨 界面展示

### 用户信息卡片
```
┌────────────────────────────────────────────────┐
│  [头像]  鱼西子                                 │
│          抖音号：84288625679  ID: 76164473...   │
│          这是我个人简介...                       │
│          关注 44  粉丝 0  获赞 0                │
└────────────────────────────────────────────────┘
```

## 🔧 技术实现

### 后端修改

1. **backend/scraper/user_scraper.py**
   - 新增 `_get_user_from_ssr()` 方法
   - 直接从 `window.SSR_RENDER_DATA` 读取用户信息
   - 优先使用 SSR 数据，失败时才使用 API 拦截
   - 避免了额外的 API 请求，更快更可靠

2. **backend/scraper/engine.py**
   - 修复验证码输入框选择器：`#button-input:not(.tnpNAdqe)`
   - 解决双输入框问题

### 前端修改

1. **frontend/src/views/Sessions.vue**
   - 新增用户信息卡片组件
   - 登录成功后自动调用 `/api/sessions/current-user` 获取用户信息
   - 展示用户头像、昵称、抖音号、ID、简介、统计数据

## 🔄 完整流程

1. 用户扫码登录
2. 输入验证码
3. 登录成功（SSE 推送 `success` 状态）
4. 前端自动调用 `/api/sessions/current-user`
5. 后端从 SSR 数据读取用户信息（或从数据库读取）
6. 前端展示用户信息卡片

## 📊 数据来源优先级

1. **SSR 数据**（`window.SSR_RENDER_DATA`）- 最快
2. **数据库**（之前采集过）- 次选
3. **API 拦截**（user/profile/self）- 兜底

## 🎯 注意事项

1. **登录成功后自动采集**：
   - 第一次登录：自动采集用户信息到数据库
   - 以后登录：直接从数据库读取

2. **错误处理**：
   - 如果用户未登录（401），不显示用户信息
   - 如果采集失败，会在日志中记录警告

3. **数字格式化**：
   - 大于 1 万显示为 "X.X万"
   - 例如：12345 → "1.2万"

## ✅ 测试方法
