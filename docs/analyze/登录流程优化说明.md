**优化时间**：2026-03-12
**优化依据**：Chrome DevTools 实际调研结果

## 一、优化概览

### 1.1 优化前问题

| 问题 | 影响 |
|------|------|
| 使用多个模糊选择器点击短信选项 | 点击成功率低，容易误触 |
| 验证码输入框选择器不准确 | 无法准确检测输入框 |
| 验证按钮类型错误（使用button） | 无法找到DIV类型的按钮 |
| 缺少"保存登录信息"处理 | 登录流程不完整 |
| 没有超时检测机制 | 验证窗口超时后无法恢复 |
| 登录成功检测不准确 | 仅依赖cookie，缺少URL检查 |

### 1.2 优化后改进

| 改进点 | 具体优化 | 效果 |
|--------|---------|------|
| **使用准确ID** | `#uc-second-verify` | 精确定位验证窗口 |
| **使用准确输入框ID** | `#button-input` | 准确检测和填入验证码 |
| **JavaScript点击** | 通过文本查找DIV按钮 | 提高点击成功率 |
| **自动处理保存对话框** | 检测并点击"保存" | 完整登录流程 |
| **超时检测** | 2分钟检测，回到扫码状态 | 自动恢复机制 |
| **URL验证** | 检查跳转到jingxuan/user | 更准确的登录判断 |

## 二、关键代码改进

### 2.1 验证窗口检测（engine.py）

```python
async def _handle_sms_consent_dialog(self, page: Page) -> bool:
    """使用准确ID #uc-second-verify 检测验证窗口"""
    verify_window = await page.query_selector('#uc-second-verify')
    if not verify_window or not await verify_window.is_visible():
        return False

    # 使用JavaScript查找并点击"接收短信验证码"
    result = await page.evaluate("""
        () => {
            const verifyWindow = document.getElementById('uc-second-verify');
            if (!verifyWindow) return false;

            const allElements = verifyWindow.querySelectorAll('*');
            for (const el of allElements) {
                const text = el.textContent || '';
                if (text.includes('接收短信验证码') && el.onclick !== null) {
                    el.click();
                    return true;
                }
            }
            return false;
        }
    """)
```

**关键点**：
- ✅ 使用 `#uc-second-verify` 精确定位
- ✅ 在窗口内查找，避免误触其他元素
- ✅ 检查 `el.onclick !== null` 确保可点击

async def fill_verify_code(self, page: Page, code: str) -> bool:

### 2.4 "保存登录信息"对话框处理（engine.py）

```python
async def _handle_save_login_dialog(self, page: Page) -> bool:
    """自动处理'保存登录信息'对话框"""
    has_dialog = await page.evaluate("""
        () => {
            const allElements = Array.from(document.querySelectorAll('*'));
            return allElements.some(el =>
                el.textContent?.includes('保存登录信息')
            );
        }
    """)

    if not has_dialog:
        return False  # 对话框不存在，跳过

    # 点击"保存"按钮
    result = await page.evaluate("""
        () => {
            const allElements = Array.from(document.querySelectorAll('*'));
            const saveBtn = allElements.find(el =>
                el.textContent?.trim() === '保存' &&
                !el.disabled &&
                (el.tagName === 'DIV' || el.tagName === 'BUTTON')
            );
            if (saveBtn) {
                saveBtn.click();
                return true;
            }
            return false;
        }
    """)
```

**关键点**：
- ✅ 先检测对话框是否存在
- ✅ 不存在时不报错，直接返回
- ✅ 支持 DIV 和 BUTTON 两种类型

### 2.5 超时检测机制（engine.py）

```python
async def wait_for_login(self) -> bool:
    """增加超时检测，自动恢复"""
    verification_window_seen = False
    verification_window_start_time = None

    for i in range(180):  # 3分钟
        # 检测验证窗口超时
        if verification_window_seen and verification_window_start_time:
            elapsed = i - verification_window_start_time
            if elapsed > 120:  # 2分钟后检查
                qr_visible = await page.query_selector('#animate_qrcode_container')
                if qr_visible and await qr_visible.is_visible():
                    logger.warning("❌ 验证窗口超时，回到扫码状态")
                    verification_window_seen = False
                    verification_window_start_time = None
```

**关键点**：
- ✅ 记录验证窗口出现时间
- ✅ 2分钟后检查是否回到扫码状态
- ✅ 自动重置状态，允许重新扫码

## 十、问题修复记录

### 10.1 误判手机号登录输入框（2026-03-13 00:20）

**问题描述**：
```
检测到验证码输入框，手机号:
检测到验证码输入框，手机号:
检测到验证码输入框，手机号:
```

扫码后一直卡在检测输入框，原因是：
- 扫码页面左侧是二维码，右侧是"手机号登录"
- 右侧的输入框被误判为验证码输入框
- 导致无法进入正确的验证流程

**根本原因**：
- 原代码使用 `input[placeholder*='验证码']` 全局查找
- 会匹配到页面右侧的"请输入手机号"输入框
- 没有限定在验证窗口 `#uc-second-verify` 内查找

**修复方案**：
```python
# 修复前：全局查找输入框（错误）
el = await page.query_selector("input[placeholder*='验证码']")

# 修复后：只在验证窗口内查找（正确）
verify_window = await page.query_selector('#uc-second-verify')
if not verify_window or not await verify_window.is_visible():
    return None  # 验证窗口不存在，直接返回

el = await page.query_selector('#uc-second-verify #button-input')
```

**关键改进**：
1. ✅ 先检查验证窗口是否存在
2. ✅ 只在验证窗口内查找输入框
3. ✅ 使用精确ID选择器 `#uc-second-verify #button-input`
4. ✅ 验证输入框的placeholder是"验证码"相关

**修复文件**：
- `backend/scraper/engine.py` - `detect_verify_code_input()`
- `backend/scraper/engine.py` - `fill_verify_code()`
- `backend/api/sessions.py` - `login_stream()`

**测试验证**：
- 扫码后不再误报检测到输入框
- 只有在验证窗口出现时才检测输入框
- 正确进入 verify 状态并显示手机号
